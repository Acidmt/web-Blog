语法规范与简介

关键字大写，空格缩进，分号结尾；

sql分为：DDL（用来定义数据库对象：数据库，表，列等。）

DML（用来对数据库中表的数据进行增删改。）

DQL（用来查询数据库中表的记录（数据等）。）

DCL （用来定义数据库的访问权限及安全级别，及创建用户等。）

一.插入记录

  1. 值与字段必须对应，个数相同，类型相同。
  2. 值的数据大小必须在字段的长度范围内。
  3. 除了数值类型外，其他字段类型的值必须用单引号（双引号）引起。
  4. 如果要插入空值，可以不写字段，或插入NULL。

条件查询

注意前面讲到的查询都是将所有数据查询出来，但更多的时候我们希望得到的是满足条件的数据，接下来会讲述该方法。

语法格式：SELECT 字段名 FROM 表名 WHERE 条件;

大致操作是：取出表中的数据，满足条件的记录就返回，不满足条件的记录不返回。

一.比较运算符

<=、>=、<>(不等于)或！=

xxxxxxxxxx
1
1

xxxxxxxxxx

1

二.逻辑运算符

AND 与(都满足则为真)

OR 或(满足一个则为真)

NOT 非(不满足则为真)

xxxxxxxxxx
1
1
SELECT * FROM student age>18 AND math>90;               #查询年龄大于18岁并且数学成绩大于90分的学生
xxxxxxxxxx
2
1
SELECT * FROM student age>18 OR math>90 OR English = '四级';              
2
#查询年龄大于18岁或者数学成绩大于90分的学生,或英语为四级的学生

SELECT 字段名 FROM WHERE 字段 IN(8,9);

IN里面的每个数据都会作为一次条件，只要满足条件的数据就会输出

xxxxxxxxxx

_: 表示一个字符

4.排序

通过ORDER BY来进行简单排序，且不会影响数据库中数据的顺序

语法格式：SELECT 字段名 FROM 表名 WHERE 字段=值 ORDER BY 字段名 ASC/DESC;

ASC: 升序(可不写)

DESC：降序

4.1单列排序

单列排序就是使用一个字段排序

xxxxxxxxxx
1
1
SELECT * FROM student WHERE age<=20 ORDER BY age ASC;   #查询表中年龄小于等于35岁的人，并按照年龄进行升序排序
4.2组合排序

组合排序就是先按第一个字段进行排序，如果第一个字段相同，才按第二个字段进行排序，之后以此类推。

语法格式：SELECT 字段名 FROM 表名 WHERE 字段=值 ORDER BY 字段名1 ASC/DESC，字段名2 ASC/DESC;

xxxxxxxxxx
2
1
SELECT * FROM student WHERE age<=20 ORDER BY age ASC,math DESC;
2
#查询表中年龄小于等于20岁的人，如果年龄相同，则按数学成绩进行降序排序

三.聚合函数

之前的查询都是横向查询，他们都是根据条件一行一行的进行判断，而使用聚合函数查询则是纵向查询，他是对一列的值进行计算，然后返回一个结果值。注意：聚合函数会忽略空值

五个常用的聚合函数：

COUT: 统计指定列的数据，为NULL的忽略不计

SUM: 计算指定列的数值和，如果不是数值类型，那么计算结果为0

MAX： 计算指定列的最大值

MIN： 计算指定列的最小值

AVG： 计算指定列的平均值，如果不是数值类型，那么计算结果为0

语法格式：SELECT COUNT(字段名) FROM 表名

xxxxxxxxxx
1
1
SELECT COUNT(*) FROM student;                   #查询表中的总人数
xxxxxxxxxx
1
1
SELECT MAX(math) FROM student;                  #查询表中数学成绩最高的分数

四.分组

分组查询是使用 GROUP BY 语句对查询信息进行分组，相同的数据作为一组

语法格式： SELECT 字段1，字段2...FROM 表名 GROUP BY 分组字段 HAVING 条件;

xxxxxxxxxx
1
1
SELECT COUNT(math) sex FROM student GROUP BY sex;       #查询表中男女生的数学成绩总和，并分别输出   
xxxxxxxxxx
1
1
SELECT sex COUNT(*) FROM student WHERE age>18 GROUP BY sex; #查询表中大于18岁的人，并按性别分组，统计每组中的人数
xxxxxxxxxx
2
1
SELECT sex COUNT(*) FROM student WHERE age>18 GROUP BY sex HAVING COUNT(*)>2;
2
#查询表中大于18岁的人，并按性别分组后统计每组中的人数，但只显示统计数据大于2的数据

HAVING和WHERE的区别

  1. HAVING是在分组后对数据进行过滤
  2. WHERE是在分组前对数据进行过滤
  3. HAVING后面可以使用聚合函数
  4. WHERE后面不可以使用聚合函数

五.LIMIT语句

LIMIT是限制的意思，所以LIMINT的作用就是限制查询记录的条数。

语法格式： LIMIT offset，length; 或者 LIMIT length;

offset是指偏移量，可以认为是跳过的记录数量，默认为0

length是指需要显示的总记录数

xxxxxxxxxx
1
1

数据库约束

对表中的数据进行进一步的限制，保证数据的正确性、有效性和完整性。

约束种类：

  1. PRIMARY KEY: 主键
  2. UNIQUE：唯一
  3. NOT NULL：非空
  4. DEFAULT：默认
  5. FOREIGN KEY ：外键(不为重点)

一.主键

1.1 主键的作用

用来唯一标识一条记录，每个表都应该有一个主键，并且每一个表只能有一个主键。

像有些记录除了唯一的id不一样，剩下的数据都一样(如：名字，年龄性别，出生日期都一样的两组数据)，那么就没法区分这些数据，从而造成了数据库记录不唯一，这样就不方便管理数据。

注意：通常情况下将一张表中的ID作为主键。主键是给数据库和程序使用的，所以主键有没有含义没有关系，只要不重复，非空也可以。

1.2 创建主键

主键特点：

  1. 主键必须包含唯一的值
  2. 主键列不能包含BULL值

格式：字段名 字段类型 PRIMARY KEY;
